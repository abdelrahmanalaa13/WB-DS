Design System Rethink

Quiet Intelligence

A design system that feels like a calm, intelligent assistant. Not flashy. Not boring. Purposeful clarity for AI-native enterprise software.

Philosophy

The DS draws from Apple's clarity-deference-depth philosophy, the Linear aesthetic movement (monochromatic + single accent), and Dieter Rams' principle that good design is as little design as possible. Every visual decision earns its place.

01
Typography Creates Hierarchy
Size and weight establish structure. Color is reserved for meaning, not decoration. A page should be scannable in grayscale.
02
Color Means Something
Brand accent for action. Red for danger. Green for success. Purple for AI. No color without semantic purpose. Surfaces stay neutral.
03
Restraint Over Novelty
Each token earns its place. No "nudge" values, no variant explosion. If a token can't justify its existence, it doesn't exist.
04
Content First, Chrome Never
UI elements recede. Data, conversations, and workflows are the foreground. The system defers to the content it presents.
05
Density With Breathing Room
ERP demands density. AI demands space. Two density modes solve this: Comfortable for conversations, Compact for data tables.
06
Motion With Purpose
Spring physics for natural feel. Every animation is interruptible. Reduced-motion always respected. If it doesn't aid comprehension, it doesn't move.

Aesthetic Position

This DS sits at the intersection of three movements: Linear Aesthetic (dark-first, monochromatic surfaces, subdued chrome, content-forward), Apple Clarity (opacity-based hierarchy, concentric radius, spring motion, semantic color), and Bauhaus Function (form follows function, no decoration without purpose). The result is enterprise software that feels intelligent, not corporate.

Color System

Color is earned, not given. Neutral surfaces dominate. Brand accent appears at moments of action. Status colors carry meaning. Content hierarchy is achieved through opacity-equivalent values, not hue.

Surface Hierarchy

Five surface levels replace the previous three. Each surface has a clear use case. In dark mode, raised surfaces get lighter (not darker) to simulate elevation toward a light source.

Content Hierarchy

Inspired by Apple's opacity-based label system. Four levels of text emphasis, all contrast-verified against their surface. Naming shifts from "on-surface" (M3 jargon) to "content" (plain English).

Borders

Three border levels. Subtle for dividers, default for containers, strong for emphasis. All derived from content colors at reduced opacity equivalents.

Brand Accent

The logo color expressed across four emphasis levels: filled (bold actions), tonal (secondary actions), outlined (tertiary), text (minimal). WideBot uses orange. Hulul uses indigo blue.

Status Colors

Five semantic roles (danger, warning, success, neutral, ai) each with four emphasis levels. Always paired with icons per DS_RULES.md Section 9. Status colors are tuned per brand to meet their respective WCAG target.

Typography

The biggest gap in the previous system. A complete type scale based on Rubik (Latin + Arabic natively in one font), with 14px body text as baseline. Negative letter-spacing at body sizes, positive at display (Apple HIG technique). JetBrains Mono for code and data.

Spacing

4px base grid, Tailwind-inspired naming (no commas in token names). Half-steps use dash notation: --space-0-5, --space-1-5, etc.

Shape

Simplified from 11 confusing tokens to 8 clean ones. WideBot uses sharper radii (enterprise authority). Hulul uses rounder radii (approachable). Both follow the concentric radius rule: inner = outer - padding.

Motion

Spring physics as the default model (Apple HIG). Cubic-bezier easing provided as CSS fallback. All animations must be interruptible and respect prefers-reduced-motion.

Touch Targets

Minimum 44x44px tap area (WCAG AAA). Visual element can be smaller. The invisible tap zone extends via padding or negative margin. This is non-negotiable for both brands.
