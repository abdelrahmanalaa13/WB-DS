Prioritized Enhancements

P0 CriticalEffort: LIBM Carbon

Data Visualization System

Zero charting components. An ERP without charts is incomplete. Evaluate Carbon Charts (D3.js) or build a custom chart component set (line, bar, pie, scatter, heatmap) with token-driven theming.

P0 CriticalEffort: MIBM Carbon

Accessibility Guild + Screen Reader Audit

We claim WCAG AAA but lack formal governance. Establish a cross-functional a11y team. Audit all 37 components against NVDA, JAWS, VoiceOver. Document testing matrix.

P0 CriticalEffort: MAnthropic + OpenAI

Artifact/Code Renderer Component

AI Chat lacks live rendering of generated code. Build an artifact renderer that supports React, HTML, SVG, and Mermaid output inline in conversation.

P1 HighEffort: LMaterial 3 + Fluent 2

3-Tier Token Architecture

Add a reference token layer under our semantic tokens. ref (brand primitives) / sys (cross-cutting) / comp (component-local). Scales better as brands multiply.

P1 HighEffort: MApple HIG + Material 3

Spring/Physics Animations

Add spring tokens (--motion-spring-damping, --motion-spring-frequency) alongside current curves. Migrate modals, dropdowns, and sheets to spring motion for natural feel.

P1 HighEffort: MApple HIG

Icon System Expansion (500+)

36 mapped HugeIcons is limiting. Expand mappings to 500+ with weight/scale configuration per icon. Or evaluate variable icon systems.

P1 HighEffort: XLIBM Carbon

Chart Components (Line, Bar, Pie, Scatter)

Even with a data viz system, individual chart components need design tokens, accessibility, RTL support, and responsive behavior. Full implementation is XL effort.

P2 MediumEffort: LFluent 2

Slot-Based Composition

Move from prop-only to named slots (icon, label, description, actions). Reduces variant explosion and improves composability.

P2 MediumEffort: SApple HIG

Translucency + Backdrop Blur Tokens

Add frosted glass surface tokens (--surface-glass-bg, --surface-glass-blur) for overlays, panels, and floating elements.

P2 MediumEffort: MIBM Carbon

Color Palette Expansion (8 families x 10 values)

Our palette may be sparse for data visualization and illustration. Expand to Carbon's model: 8 color families x 10 values per theme.

P2 MediumEffort: SIBM Carbon

AI Explainability Components

Add "AI-generated" badges, confidence indicators, and explainability popovers. Essential for transparent AI in enterprise ERP contexts.

P2 MediumEffort: MFluent 2

Publish ds-copilot-ui Package

Extract AI Chat, reasoning traces, and artifact renderer into a standalone package. Enables faster AI feature adoption across products.

P3 Nice-to-haveEffort: XLIBM Carbon

Web Components Port

Framework-agnostic layer for long-term flexibility. Enables Vue, Svelte, and vanilla JS adoption alongside Angular.

P3 Nice-to-haveEffort: MOpenAI

Memory System in Copilot

Auto-learn user preferences (brand, layout, components used frequently). Reduces friction for repeat users.

P3 Nice-to-haveEffort: SOpenAI

Reasoning Traces UI Component

Collapsible "thinking" component showing AI decision logic. Builds trust and aids onboarding.
