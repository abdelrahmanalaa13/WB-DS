4. XSmall Size: Slot Usage in Compound Components

The M3 Kit provides XSmall across Buttons and Icon Buttons. XSmall should NOT be used as a standalone CTA but IS valuable as a nested element inside larger compounds:

Compound ComponentXSmall Slot UsageExample
Cards Action Slot Overflow menu trigger, quick-action icon buttons in card headers
App Bars / Headers Trailing Actions Notification bell, settings gear, user menu trigger
Table Rows Inline Actions Edit, delete, expand row actions
List Items Trailing Icon Favorite toggle, status indicator, action trigger
Chat Messages Reaction Bar Emoji reactions, copy, retry, thumbs up/down
Chips Remove Action X button to dismiss a chip/tag
Tabs / Toolbar Dense Controls Close tab button, toolbar toggle icons
XSmall Verdict

Keep XSmall exclusively for Icon Buttons (not text buttons). Map its touch target to minimum 32x32px for accessibility. In your token system, define --button-icon-xs-size: 24px with --button-icon-xs-touch-target: 32px. Never expose XSmall as a standalone button option to product teams.

5. Generation Pipeline

ONE-TIME Upload globals.css + DS_RULES.md to Lovable project

1. Claude generates strict Lovable prompt (per wave)
  ↓ human reviews prompt (~1 min)
2. Human sends prompt to Lovable (1-shot generation)
  ↓ human reviews visually (~2-5 min)
3. Human approves, Claude generates MDs + Figma sync
  ↓ human reviews (~2-5 min)
4. Next component

Lovable Kit

Three files power the pipeline: globals.css (all DS tokens as CSS vars, 2100+ lines), DS_RULES.md (component architecture, state layers, motion, responsive rules, mobile adaptations, accessibility, RTL, AI patterns), and LOVABLE_PROMPT.md (master prompt covering all 30+ component pages across 5 waves with exact variant specifications). Claude accesses Lovable output via GitHub sync (Lovable pushes to GitHub, Claude reads the repo) or the community lovable-mcp-server (read-only component analysis).
